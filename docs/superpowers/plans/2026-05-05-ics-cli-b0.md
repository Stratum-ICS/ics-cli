# B0 — Local hybrid store + CLI Implementation Plan (Rust)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Cargo package **`ics-cli`** whose **`[[bin]]`** target installs as **`ics`**, implementing `init`, `commit`, `log`, `diff`, `status`, and optional path restore from `HEAD`, using **hybrid storage** per [`2026-05-06-ics-cli-design.md` §3.1.1](../specs/2026-05-06-ics-cli-design.md).

**Architecture:** Library crate (`src/lib.rs`) holds repo/types/store/objects; thin `src/main.rs` wires **clap** subcommands. Same `.ics/` layout as the spec (blobs + trees on disk, SQLite graph/refs).

**Tech Stack:** Rust **2021**, **clap**, **rusqlite**, **serde** / **serde_json**, **sha2**, **hex**, **anyhow**, **thiserror**, **tempfile** (dev). No HTTP / Stratum in B0.

**Files:**
- Create: `src/paths.rs`, `src/objects.rs`
- Create: `tests/objects.rs` or `#[cfg(test)]` in `objects.rs`

**Conventions:**
- `blob_hash(data: &[u8]) -> [u8; 32]` using **sha2::Sha256**; display as **lowercase hex**.
- Tree manifest: `BTreeMap<String, String>` serialized with **serde_json** using **sorted keys** — use `BTreeMap` + `serde_json::to_vec` with `serde_json::Map` from sorted iteration; hash = SHA-256 of those UTF-8 bytes.

### Task 3: SQLite store

**Files:**
- Create: `src/store.rs`

**Schema (exact SQL):**

```sql
PRAGMA foreign_keys = ON;

CREATE TABLE IF NOT EXISTS commits (
  commit_id TEXT PRIMARY KEY NOT NULL,
  tree_id TEXT NOT NULL,
  parent_commit_ids TEXT NOT NULL,
  message TEXT NOT NULL,
  author TEXT NOT NULL,
  created_at INTEGER NOT NULL
);

CREATE TABLE IF NOT EXISTS refs (
  name TEXT PRIMARY KEY NOT NULL,
  commit_id TEXT NOT NULL
);
```
