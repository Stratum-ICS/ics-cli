# B0 — Local hybrid store + CLI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Python package **`ics-cli`** with console script **`ics`** implementing `init`, `commit`, `log`, `diff`, `status`, and optional `checkout`-style revert of paths from `HEAD`, using **hybrid storage**: SHA-256 blobs + tree JSON files under `.ics/objects/`, commit DAG + refs in **SQLite** per [`2026-05-06-ics-cli-design.md` §3.1.1](../specs/2026-05-06-ics-cli-design.md).

**Architecture:** Single workspace root holds `.ics/` metadata and user markdown outside `.ics/`. Commits reference a **tree** object (JSON map `relative_path → blob_sha256`). Blobs store raw file bytes. Refs table maps `HEAD` and branch names to commit ids. Default branch name **`main`**.

**Tech Stack:** Python ≥3.10, Typer, stdlib `sqlite3`, `unittest`, `tomllib` / setuptools packaging. No Stratum / HTTP in B0.

### Task 2: Paths + blob/tree objects

**Files:**

- [ ] **Step 1: Write tests for blob and tree hashing**

```python
import tempfile
import unittest

from ics_cli.objects import blob_hash, read_blob, tree_hash, write_blob, write_tree


class TestObjects(unittest.TestCase):
    def test_blob_hash_stable(self):
        self.assertEqual(
            blob_hash(b"hello"),
            "2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824",
        )

    def test_blob_roundtrip(self):
        with tempfile.TemporaryDirectory() as tmp:
            h = write_blob(tmp, b"hello")
            self.assertEqual(read_blob(tmp, h), b"hello")

    def test_tree_hash_stable(self):
        m = {"notes/a.md": blob_hash(b"x")}
        self.assertEqual(len(tree_hash(m)), 64))
```

Implement `write_blob(ics_root, data) -> hex`, `read_blob(ics_root, hex) -> bytes`, `write_tree(ics_root, path_to_blob_hex: dict) -> tree_hex`, `tree_hash(...)` in Step 2.

**Object conventions:**
- `blob_hash(data: bytes) -> str` — SHA-256 hex lowercase, 64 chars.
- Tree manifest: UTF-8 JSON object with keys **sorted lexicographically**, values 64-char hex blob hashes. Use `json.dumps(obj, sort_keys=True, separators=(",", ":"))` then encode UTF-8; `tree_hash` = SHA-256 of those bytes.
- File paths: `.ics/objects/blobs/{h[:2]}/{h[2:]}` and `.ics/objects/trees/{h[:2]}/{h[2:]}`.

### Task 3: SQLite store (commits + refs)

**Files:**
- Create: `src/ics_cli/store.py`
- Test: `tests/test_store.py`

- [ ] **Step 1: Schema (exact SQL)**

```sql
PRAGMA foreign_keys = ON;

CREATE TABLE IF NOT EXISTS commits (
  commit_id TEXT PRIMARY KEY NOT NULL,
  tree_id TEXT NOT NULL,
  parent_commit_ids TEXT NOT NULL,
  message TEXT NOT NULL,
  author TEXT NOT NULL,
  created_at INTEGER NOT NULL
);

CREATE TABLE IF NOT EXISTS refs (
  name TEXT PRIMARY KEY NOT NULL,
  commit_id TEXT NOT NULL
);
```

`parent_commit_ids` stores JSON array of hex strings, e.g. `[]` or `["abc..."]`. Single parent for B0 linear history.

### Task 5: CLI — `init`, `status`, `commit`, `log`, `diff`

**Files:**
- Modify: `src/ics_cli/cli.py`
- Modify: `tests/test_cli.py`

**Behavior:**
- `ics init` — creates `.ics/` tree with empty DB and no commits; sets optional default branch `main` ref absent until first commit (or create orphan — **choose**: first `commit` creates `HEAD`/`main`; `status` before first commit shows all `.md` as untracked/new).
- `ics status` — shows modified / added / deleted vs `HEAD` tree (if no HEAD, all tracked md are “new”).
- `ics commit -m "msg" [--author NAME]` — requires prior `init`; runs `make_commit`.
- `ics log` — walks parents from `HEAD`, prints short hash + first line of message (newest first or git order — document choice).
- `ics diff` — unified diff of working tree vs `HEAD` tree (stdlib `difflib`).
