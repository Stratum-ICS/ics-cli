# B1 — Stratum HTTP integration Implementation Plan (Rust)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend **`ics`** so an authenticated user can sync markdown with Stratum notes and proposals (`POST /api/proposals`), keeping Stratum IDs in **YAML frontmatter** or `.ics/stratum-index.json` per design §3.2.

**B1+ module layout (extend `lib.rs`):** `pub mod stratum;` + `src/stratum/mod.rs` re-exporting `client`, `routes`, `errors`; new top-level `config`, `auth_store`, `frontmatter`, `identity` (or nest under a `bridge` module — pick one style in Task 1 and keep consistent).

**Architecture:** **`StratumClient`** wraps **`reqwest::Client`** with base URL + bearer token. Routes pinned from OpenAPI (see Task 0). Use **`#[tokio::main] async fn main` in `src/main.rs`** and `async` client methods; call sync B0 library code with `spawn_blocking` or `std::thread` only if needed (prefer keeping B0 operations sync inside blocking sections).

**Tech Stack:** **reqwest**, **tokio**, **serde** / **serde_json**, **serde_yaml**, **rusqlite** (optional cache), **directories** or manual XDG for config dir. Tests: **`wiremock`** or **`mockito`** for HTTP stubs.

**Prerequisite:** B0 complete.

**OpenAPI policy (all milestones):** Do not commit generated files under **`.claude/tmp/`** to git. Save a **reproducible** fixture (e.g. `tests/fixtures/stratum-openapi.json` or `docs/third_party/stratum-openapi@<git-sha>.json`) **or** store only a **SHA-256 of `openapi.json` in a comment** at the top of `routes.rs` plus the Stratum **git tag/SHA** the pin came from. B1 Task 0 Step 4 should **`git add`** that fixture or rely on checksum-only — never the ephemeral curl path by default.

### Task 2: Stratum client + wiremock

**Files:**
- Create: `src/stratum/client.rs`, `src/stratum/errors.rs`
- Modify: `src/stratum/mod.rs` (from Task 0 — export `routes`, `client`, `errors`)
- Create: `tests/stratum_client.rs`
- Modify: `Cargo.toml` — add **runtime:** `reqwest = { version = "0.12", features = ["json"] }`, `tokio = { version = "1", features = ["rt-multi-thread", "macros"] }`; **dev:** `wiremock = "0.6"` (tests need **`tokio` test macros** — ensure `[dev-dependencies]` include `tokio` with `macros` if not inherited).

- [ ] **Step 1: Example test** — wiremock **`path()` matcher must use `routes::POST_PROPOSALS`** (import module), **not** a duplicated string literal.

```rust
use wiremock::{Mock, MockServer, ResponseTemplate};
use wiremock::matchers::{method, path};

use ics_cli::stratum::routes;

#[tokio::test]
async fn post_proposal_ok() {
    let srv = MockServer::start().await;
    Mock::given(method("POST"))
        .and(path(routes::POST_PROPOSALS))
        .respond_with(ResponseTemplate::new(200).set_body_json(serde_json::json!({"id":"p1","status":"pending"})))
        .mount(&srv)
        .await;

    let client = StratumClient::new(srv.uri(), "token".into());
    let out = client.post_proposal(&serde_json::json!({
        "team_id": 1,
        "note_ids": [2],
        "rationale": "x".repeat(50)
    })).await.unwrap();
    assert_eq!(out["id"], "p1");
}
```

Adjust **`POST_PROPOSALS`** placeholder path in `routes.rs` before tests compile.
