# C — Vault-shaped sync (Stratum `/api/vaults/...`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add **vault** workflows aligned with **shipped** Stratum code (`backend/src/stratum/api/vaults.py`), especially **`POST /api/vaults/{slug}/pull`** with `confirm: false` (preview) and `confirm: true` (apply). Optional: fork/clone/tree endpoints **exactly as OpenAPI lists** — no blueprint `sync/diff` names from deprecated docs.

**Architecture:** New Typer group `ics vault ...`. Reuse B2 conflict instincts: preview shows server-returned diff summary or file list before apply. **Does not remove** B0 local commits — vault pull is an **additional** sync path per design §3.3.

**Tech Stack:** httpx client extensions; may reuse Stratum wiki [`vault-layout.md`](https://github.com/Stratum-ICS/Stratum/blob/master/wiki/vault-layout.md) for on-disk path alignment.

**Prerequisite:** B2 done; team confirms vault API stability; OpenAPI includes vault routes.
