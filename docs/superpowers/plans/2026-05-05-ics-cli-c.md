# C — Vault-shaped sync Implementation Plan (Rust)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** **`ics vault …`** subcommands aligned with shipped Stratum **`vaults.py`** / OpenAPI — primarily **`POST /api/vaults/{slug}/pull`** with `confirm: false` then `confirm: true`.

**Architecture:** Nested **clap** struct: `ics vault pull <slug> [OPTIONS]`. Client lives in **`src/stratum/vault_client.rs`**. Preview/apply **never** skip confirmation unless **`--yes`** flag documented as dangerous.

**Tech Stack:** **reqwest** + **serde**; optional **serde_json::Value** for preview payloads if schema fluctuates — tighten to structs once OpenAPI stabilizes.

**Prerequisite:** B2 done; vault routes verified on target Stratum version.

---

### Blocking checklist

- [ ] **`--yes`** on `vault pull` is destructive (applies server changes). Document that automation must pass **`--yes`** explicitly; do not default to apply in CI without review.
