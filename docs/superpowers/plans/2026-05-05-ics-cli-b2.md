# B2 — Pull server state + conflict policy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** **`ics`** can **refresh** local markdown from Stratum truth (list/read note APIs per pinned OpenAPI), update frontmatter/index, and apply an **explicit per-command conflict policy** when the working tree or local commits diverge from server content.

**Architecture:** Build on `StratumClient` from B1. Add **read path**: fetch note metadata + body for IDs known from index/frontmatter and optionally discover workspace-linked notes. Compare bytes with working tree; never silently overwrite — flags choose **take-mine**, **take-theirs**, or **write `.conflict` sidecar** (pick one default + document in `--help`).

**Tech Stack:** Same as B1; optional small **three-way merge** only if you add it — default plan avoids merge and uses explicit policies only.

**Prerequisite:** B1 complete; `routes.py` includes read/list note endpoints from Stratum OpenAPI.
