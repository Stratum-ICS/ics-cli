## Context Summary

Spin up a separate **`ics-cli`** repository that documents and will implement a CLI: **local markdown workspace** with **git-like** local history (B0–B2), then **vault-shaped** integration with Stratum (C) when server-side vault behavior is stable enough. **ICS** (teams, proposals, reviews) uses Stratum’s existing **REST API**. Stratum remains the backend of record; this repo is the CLI + integration spec only.

## Key Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|-------------------------|
| Separate repo `ics-cli` | Keeps release cycle and **crates.io** binary independent from Stratum monolith | Monorepo under Stratum |
| **B before C** | Local workflow + ICS HTTP works without full vault completeness; vault API still evolving | Vault-first CLI |
| Identity via frontmatter (preferred) | Portable with Obsidian; files carry Stratum IDs | Index-only mapping |
| Track real `POST /api/vaults/{slug}/pull` | Matches shipped Stratum; blueprint `sync/diff` is stale | Implement blueprint URLs |
